# 次回セッション: 階層パンくずとファイル管理

## 目的

現在のパンくずは、開いたフォルダ直下の `.md` ファイル一覧を切り替える最小実装である。

次回はこれを、フォルダ階層を辿れる完全なパンくずリストに拡張し、ファイル/フォルダのリネーム、削除、右クリックメニュー、並び替えまで進める。

## 現在できていること

- ハンバーガーメニューから新規ファイル、ファイルを開く、保存、別名保存、フォルダを開く操作ができる
- Tauri側で `.md` ファイルの読み書き、保存ダイアログ、フォルダ選択を扱っている
- 開いたフォルダ配下のサブフォルダと `.md` ファイルを再帰的なツリーとして一覧化できる
- パンくずを `ルート > サブフォルダ > ... > ファイル名` として表示できる
- 各フォルダ crumb のドロップダウンから子フォルダ/Markdownファイルへ移動できる
- パンくずメニューから新規ファイル、新規フォルダの作成、ファイル/フォルダのリネーム、削除、上下移動による並び替えができる
- 削除はUI確認付き。フォルダ削除は安全のため空フォルダのみ対応する
- 並び替えはプロジェクトルートの `.brew/project.json` に保存する

## 次に実装する範囲

### 1. プロジェクトツリーのデータ構造

- 実装済み。Tauri側でフォルダ階層を再帰的に読み、フォルダと `.md` ファイルを同じ `ProjectEntry` ツリー構造で返す
- 実装済み。フロント側では `ProjectFolder.children` をツリー構造として扱う
- 実装済み。現在選択中のファイルパスから、ルートフォルダからファイルまでのパンくず配列を計算する

### 2. 階層パンくず

- 実装済み。パンくずを `ルート > サブフォルダ > ... > ファイル名` として表示する
- 実装済み。各フォルダ crumb はクリック/右クリックでその階層の子フォルダ/ファイル一覧を開く
- 実装済み。ファイル crumb は現在開いているファイル名を表示する
- 長いパスでは中央の crumb を省略するか、横幅内で ellipsis する

### 3. 右クリックメニュー

- フォルダ crumb の右クリック:
  - 新規ファイル作成: 実装済み
  - 新規フォルダ作成: 実装済み
  - リネーム: 子フォルダ/ファイル一覧項目で実装済み
  - 削除: 子フォルダ/ファイル一覧項目で実装済み
- ファイル crumb / ファイル一覧項目の右クリック:
  - リネーム: 実装済み
  - 削除: 実装済み
  - 複製は必要なら後回し
- 削除は必ず確認ダイアログを出す: 実装済み

### 4. リネーム

- 実装済み。Tauri側にファイル/フォルダのリネーム command を追加する
- 実装済み。Windowsで使えない文字を拒否する
- 実装済み。既存名との衝突時はエラー表示する
- 実装済み。開いているファイルをリネームした場合は `currentFilePath` と `currentFileName` を更新する

### 5. 削除

- 実装済み。Tauri側にファイル/フォルダ削除 command を追加する
- フォルダ削除は中身がある場合の扱いを決める
  - 実装済み。最初は空フォルダのみ削除可にする
  - 再帰削除は明示確認付きで後回しでもよい
- 実装済み。開いているファイルが削除された場合は、プロジェクト内の次の `.md` を開く。なければ新規未保存状態に戻す

### 6. 並び替え

- まずは表示順の仕様を決める
  - 初期表示順は「フォルダ優先、同種内は名前昇順」
  - 実装済み。パンくずメニュー内の上下ボタンで同一フォルダ内の子要素を手動並び替えできる
  - 実装済み。保存先はプロジェクトルートの `.brew/project.json`。フォルダごとに `order` を保存する
- 今後はドラッグ操作による並び替え、リネーム時の既存 order 名更新、不要になった order エントリの掃除を検討する

## 実装時の注意

- 既存の `.md` 読み書き command を壊さない
- スニペット保存方式の分離にはまだ踏み込まない
- OSファイル操作はTauri側に寄せ、フロント側ではパス文字列を直接加工しすぎない
- 削除や上書きなど破壊的操作は、UI上で明示確認を取る
- `npm run build` と `npm run tauri:build` を通す
- UI変更後はスクリーンショットを取る
