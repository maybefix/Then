# 字下げモード 要件定義

## 1. 概要

字下げモードは、段落先頭に存在する全角スペース `U+3000` を、編集画面上では通常文字として描画せず、CSSの `text-indent` による字下げとして表示する機能である。

保存されるプレーンテキストは変更しない。

```text
　本文
```

上記テキストは、字下げモード有効時には次のように解釈する。

```text
ソース上の字下げ数: 1
表示上の字下げ量: 1ic
本文: 本文
```

字下げモード無効時は、全角スペースを通常文字としてそのまま表示する。

---

## 2. 目的

本機能の目的は次のとおりとする。

1. プロポーショナルフォント使用時でも、段落先頭の字下げ位置を安定させる
2. プレーンテキスト上の全角スペースを字下げ情報の正本として維持する
3. 独自の字下げ記法や字下げ無効記法を追加しない
4. 一般的なテキストエディタと近い入力、削除操作を維持する
5. 自動字下げと手動字下げを両立させる

---

## 3. 基本原則

### 3.1 正本

字下げ情報の正本は、段落先頭に存在する連続した `U+3000` とする。

AST上の字下げ数は、プレーンテキストを解析した結果として扱う。

```ts
type ParagraphNode = {
  type: "paragraph";
  indentCount: number;
  children: InlineNode[];
};
```

`indentCount` は隠れた書式情報ではなく、行頭の全角スペース数を構造化した値とする。

### 3.2 保存形式

保存時には、`indentCount` と同数の `U+3000` を段落先頭へ出力する。

```ts
function serializeParagraph(node: ParagraphNode): string {
  return "\u3000".repeat(node.indentCount)
    + serializeInline(node.children);
}
```

字下げモードの有効、無効によって保存内容を変更してはならない。

### 3.3 表示形式

字下げモード有効時は、行頭の字下げ用 `U+3000` を本文文字列から分離し、次のCSS相当で表示する。

```css
.editor[data-indent-mode="enabled"] .paragraph {
  text-indent: calc(var(--indent-count) * 1ic);
}
```

字下げモード無効時は、`U+3000` を本文中の通常文字として描画する。

---

## 4. 設定構成

設定画面には、次の項目を設ける。

### 4.1 行頭字下げモード

```text
行頭字下げモード
```

説明文:

```text
段落先頭の全角スペースを、フォントに応じた字下げとして表示します。
ファイル内の全角スペースは変更されません。
```

設定値:

```ts
type IndentRenderMode = "plain" | "typographic";
```

| 値             | 挙動                              |
| ------------- | ------------------------------- |
| `plain`       | 全角スペースを通常文字として表示する              |
| `typographic` | 行頭全角スペースを `text-indent` として表示する |

初期値は `typographic` とする。

### 4.2 新規段落の自動字下げ

```text
新しい段落を自動で字下げする
```

説明文:

```text
Enterで新しい段落を作成した際、段落先頭に全角スペースを追加します。
```

設定値:

```ts
autoInsertIndent: boolean;
```

初期値は `true` とする。

### 4.3 会話文の自動字下げ解除

```text
かぎ括弧で始まる段落は字下げしない
```

説明文:

```text
自動挿入された字下げの直後に「または『を入力した場合、字下げを取り消します。
手動で入力した全角スペースは削除しません。
```

設定値:

```ts
skipDialogueIndent: boolean;
```

初期値は `true` とする。

この設定は `autoInsertIndent` が有効な場合にのみ動作する。

---

## 5. 設定データ

```ts
interface IndentSettings {
  renderMode: "plain" | "typographic";
  autoInsertIndent: boolean;
  skipDialogueIndent: boolean;
}
```

編集途中の一時状態として、自動挿入された字下げかどうかを保持する。

```ts
interface PendingAutoIndent {
  paragraphId: string;
  active: boolean;
}
```

`PendingAutoIndent` はファイルへ保存しない。

---

## 6. 段落解析

段落先頭に連続する `U+3000` の数を数える。

```ts
function parseLeadingIndent(source: string): {
  indentCount: number;
  content: string;
} {
  let indentCount = 0;

  while (source[indentCount] === "\u3000") {
    indentCount++;
  }

  return {
    indentCount,
    content: source.slice(indentCount),
  };
}
```

対象は段落先頭に連続して存在する `U+3000` のみとする。

次の全角スペースは字下げとして扱わない。

```text
本文中の　空白
```

```text
本文末尾の　
```

## 7. 表示要件

### 7.1 字下げモード有効時

次のソース:

```text
　本文
```

表示結果:

```text
1ic分字下げされた本文
```

次のソース:

```text
　　本文
```

表示結果:

```text
2ic分字下げされた本文
```

CSS:

```css
.paragraph {
  --indent-count: 0;
  text-indent: calc(var(--indent-count) * 1ic);
}
```

### 7.2 字下げモード無効時

行頭の `U+3000` を通常文字としてDOMへ出力する。

```text
　本文
```

フォントが持つ実際の全角スペース幅で表示する。

### 7.3 モード切り替え

字下げモードを切り替えても、次を変更してはならない。

* ソーステキスト
* AST上の文字オフセット
* Undo履歴
* ファイルの変更状態
* 保存済みスナップショット

表示方法だけを変更する。

---

## 8. Enter入力

### 8.1 自動字下げ有効

段落末尾でEnterを押した場合、新しい段落を作成し、先頭へ `U+3000` を1つ追加する。

```text
　前の段落。|
```

Enter後:

```text
　前の段落。
　|
```

新しい段落は一時的に「自動字下げ状態」とする。

```ts
pendingAutoIndent = {
  paragraphId: newParagraph.id,
  active: true,
};
```

### 8.2 自動字下げ無効

新しい段落に全角スペースを追加しない。

```text
　前の段落。|
```

Enter後:

```text
　前の段落。
|
```

### 8.3 段落途中での分割

段落途中でEnterを押した場合は、現在の字下げを機械的に複製しない。

初版では、通常の新規段落規則を適用する。

```text
　前半|後半
```

Enter後:

```text
　前半
　後半
```

自動字下げが無効なら次の結果とする。

```text
　前半
後半
```

---

## 9. 会話文判定

### 9.1 対象文字

初版で判定対象とする文字は次の2文字とする。

```text
「
『
```

### 9.2 動作条件

次の条件をすべて満たした場合のみ、自動挿入された字下げを削除する。

1. `autoInsertIndent` が有効
2. `skipDialogueIndent` が有効
3. 対象段落が自動字下げ状態
4. 自動挿入後、まだ本文文字が入力されていない
5. 最初の入力文字が `「` または `『`

入力前:

```text
　|
```

`「`を入力:

```text
「|
```

### 9.3 手動字下げの保護

ユーザーが明示的に全角スペースを入力した場合、後続文字が `「` または `『` でも削除しない。

```text
　「本文」
```

この段落は字下げありとして維持する。

### 9.4 自動状態の終了

次のいずれかが発生した時点で、自動字下げ状態を終了する。

* `「` または `『` 以外の文字が入力された
* キャレットが別段落へ移動した
* 文字列の貼り付けが行われた
* UndoまたはRedoが実行された
* 段落の構造が変更された
* ファイルが保存、再読み込みされた

---

## 10. Backspace操作

### 10.1 基本動作

字下げモード有効時、表示上は行頭の `U+3000` がDOM本文に存在しないため、Backspaceを自前処理する。

次の条件を満たす場合:

1. キャレットが段落本文の先頭にある
2. 選択範囲がない
3. `indentCount > 0`

Backspaceによって `indentCount` を1減らす。

```text
　本文
```

本文先頭でBackspace:

```text
本文
```

### 10.2 複数字下げ

```text
　　本文
```

本文先頭でBackspaceを1回:

```text
　本文
```

さらに1回:

```text
本文
```

### 10.3 字下げがない場合

`indentCount === 0` の場合は、通常の段落結合処理へ移る。

```text
前段落
|後段落
```

Backspace:

```text
前段落後段落
```

### 10.4 字下げモード無効時

ブラウザまたはエディタの通常文字削除処理を使用する。

ただし、AST正本との整合性を保つため、最終的な削除結果は通常の編集トランザクションへ変換する。

---

## 11. Delete操作

本文先頭にキャレットがある状態でDeleteを押しても、字下げは削除しない。

字下げは本文より前に存在するため、削除操作はBackspaceに限定する。

行頭全角スペースそのものを選択している場合は、選択削除として処理する。

---

## 12. Space入力

### 12.1 全角スペース

段落本文先頭で `U+3000` が入力された場合、`indentCount` を1増やす。

```text
|本文
```

全角スペース入力後:

```text
　|本文
```

字下げモード有効時は表示上の `text-indent` が1ic増える。

### 12.2 半角スペース

半角スペース `U+0020` は字下げとして解釈しない。

通常文字として本文へ追加する。

### 12.3 本文途中

本文途中で入力された `U+3000` は通常文字として扱う。

```text
本文|続き
```

全角スペース入力後:

```text
本文　|続き
```

`indentCount` は変更しない。

---

## 13. 貼り付け

貼り付けられた各段落について、行頭に連続する `U+3000` を解析する。

```text
　第一段落
「第二段落」
　　第三段落
```

解析結果:

```ts
[
  { indentCount: 1, content: "第一段落" },
  { indentCount: 0, content: "「第二段落」" },
  { indentCount: 2, content: "第三段落" },
]
```

貼り付け時には、会話文判定による自動削除を行わない。

貼り付けられた文字列をそのまま尊重する。

---

## 14. UndoとRedo

字下げに関する次の操作は、通常の編集操作としてUndo履歴へ記録する。

* 自動字下げの挿入
* 会話文判定による自動字下げ削除
* Backspaceによる字下げ削除
* 全角スペース入力による字下げ追加
* 貼り付けによる字下げ追加

Enterと自動字下げ挿入は、原則として1つのトランザクションにまとめる。

```text
Enter
+
U+3000の自動挿入
```

Undoを1回実行すると、新しい段落作成自体が取り消される。

`「`入力時の自動字下げ削除と文字入力も、1つのトランザクションにまとめる。

---

## 15. キャレットとオフセット

ASTとソース上では、行頭の `U+3000` も文字オフセットとして数える。

```text
　本文
```

ソース上のオフセット:

```text
0: U+3000
1: 本
2: 文
```

表示上の本文先頭キャレットは、ソース上ではオフセット1に相当する。

字下げモード有効時は、表示オフセットとソースオフセットを変換する。

```ts
function sourceOffsetToViewOffset(
  sourceOffset: number,
  indentCount: number
): number {
  return Math.max(0, sourceOffset - indentCount);
}
```

```ts
function viewOffsetToSourceOffset(
  viewOffset: number,
  indentCount: number
): number {
  return viewOffset + indentCount;
}
```

ただし、選択範囲が字下げ部分を含む場合に備え、字下げ位置を表す論理キャレット位置を別途扱う。

```ts
type ParagraphPosition = {
  paragraphId: string;
  sourceOffset: number;
};
```

DOMオフセットを正本にしてはならない。

---

## 16. 選択操作

### 16.1 通常選択

本文文字の選択は従来どおり行う。

### 16.2 段落全体の選択

段落全体を選択した場合は、行頭の `U+3000` も選択対象に含める。

コピー結果:

```text
　本文
```

### 16.3 本文先頭からの選択

本文先頭から右方向へ選択した場合、初期状態では字下げを含めない。

段落全体選択、行選択、全選択の場合のみ字下げを含める。

---

## 17. コピーと切り取り

コピー、切り取り時には、ASTからプレーンテキストをシリアライズする。

字下げモード有効時でも、行頭の `U+3000` をクリップボードへ出力する。

```text
表示:
  1ic字下げされた本文

コピー結果:
　本文
```

切り取り時は、選択範囲に字下げが含まれている場合のみ `indentCount` を変更する。

---

## 18. 検索と置換

検索対象には、ソース上の `U+3000` を含める。

全角スペース検索によって、字下げ部分も検索可能とする。

置換によって行頭の `U+3000` が削除された場合、`indentCount` を再計算する。

例:

```text
検索: ^　
置換: 空文字
```

結果:

```text
全段落の字下げを削除
```

正規表現検索を実装する場合、段落先頭の字下げを通常の文字列として扱う。

---

## 19. 記法との関係

字下げは独自記法として追加しない。

既存の段落寄せ記法などが先頭に存在する場合は、保存形式上の規則を別途定める必要がある。

初版では次の順序とする。

```text
U+3000
ブロック記法
本文
```

例:

```text
　[(al:center)]本文
```

解析順:

1. 行頭全角スペースを字下げとして解析
2. ブロック記法を解析
3. インライン記法を解析

字下げ数の判定時には、記法を飛ばして可視文字を探さない。

行頭の物理的な `U+3000` のみを字下げ情報とする。

---

## 20. ルビ、傍点、縦中横との関係

字下げは段落全体のレイアウトとして適用する。

次の機能は字下げ量の計算に影響させない。

* ルビ
* 傍点
* 縦中横
* インライン装飾
* コメント
* 検索ハイライト
* スペルチェック表示

```css
text-indent: calc(var(--indent-count) * 1ic);
```

は段落コンテナへ適用し、個別インラインノードへ適用しない。

---

## 21. フォント変更

`1ic` は現在のフォントのCJK文字送り幅を基準とする。

フォント変更時には、字下げの実寸も自動的に変化する。

初版ではフォント別の補正値を設けない。

将来、特定フォントで視覚的なずれが確認された場合に備え、内部的には字下げ単位をCSS変数にする。

```css
.editor {
  --indent-unit: 1ic;
}

.paragraph {
  text-indent:
    calc(var(--indent-count) * var(--indent-unit));
}
```

---

## 22. 禁則処理との関係

字下げは段落の先頭位置を変更するだけとし、禁則処理の対象文字には含めない。

行頭禁則の判定は、字下げ用 `U+3000` を除いた最初の可視文字から開始する。

```text
　、本文
```

禁則判定対象の先頭文字は `、` とする。

---

## 23. 均等割り付けとの関係

字下げモードは、既存の均等割り付けと併用可能とする。

```css
.paragraph {
  text-indent: calc(var(--indent-count) * 1ic);
  text-align: justify;
  text-justify: inter-character;
}
```

字下げ部分には均等割り付けを適用しない。

行頭全角スペースを通常の本文文字としてDOMへ残さないことで、均等割り付けによる空白幅の変動を避ける。

---

## 24. 対象外

初版では次を対象外とする。

* 行頭半角スペースを字下げとして扱う機能
* タブ文字による字下げ
* フォントごとの字下げ補正
* 小数単位の字下げ
* 負の字下げ
* ぶら下げインデント
* 段落全体のインデント
* 引用ブロック用インデント
* 原稿用紙グリッドとの連携
* AIによる会話文判定
* `「`と`『`以外の会話開始文字判定
* 保存時の自動整形
* 既存ファイルへの一括字下げ追加

---

## 25. エラー処理

ASTの `indentCount` とソース文字列が一致しない場合は、ソーステキストを優先して再解析する。

`indentCount` は0以上の整数に制限する。

```ts
indentCount = Math.max(0, Math.floor(indentCount));
```

異常値が存在しても、保存時に意図しない全角スペースを追加してはならない。

---

## 26. 受け入れ条件

### 表示

* 行頭に全角スペースが1つある段落が、字下げモード有効時に1ic字下げされる
* 行頭に全角スペースが2つある段落が、2ic字下げされる
* 本文途中の全角スペースは通常文字として表示される
* モード切り替えでファイル内容が変更されない
* フォント変更後も字下げが現在のフォントに追従する

### 入力

* 自動字下げ有効時、Enter後に全角スペースが追加される
* 自動字下げ無効時、Enter後に全角スペースが追加されない
* 自動字下げ後に`「`を入力すると字下げが削除される
* 手動入力した全角スペースは、`「`入力時にも削除されない
* 本文先頭で全角スペースを入力すると字下げ数が増える

### 削除

* 本文先頭でBackspaceを押すと字下げが1段階減る
* 複数字下げはBackspaceごとに1つずつ減る
* 字下げが0の状態では通常の段落結合が行われる
* Undoで字下げ削除を元に戻せる

### 保存

* 字下げモード有効、無効の双方で同一のプレーンテキストが保存される
* コピー時に行頭全角スペースが維持される
* 再読み込み後も字下げ数が維持される
* 他のテキストエディタで開いても、行頭全角スペースを確認できる

---

## 27. 実装優先順位

### 第1段階

* 行頭 `U+3000` の解析
* `indentCount` のAST化
* `text-indent: n ic` による表示
* 設定による表示切り替え
* 保存と再読み込み

### 第2段階

* 本文先頭のBackspace処理
* 全角スペース入力処理
* コピー、貼り付け
* Undo、Redo

### 第3段階

* Enter時の自動字下げ
* 会話文開始時の自動解除
* 自動入力と手動入力の区別

### 第4段階

* 検索、置換との統合
* 選択範囲の詳細調整
* 記法との統合テスト
* IME入力時の挙動確認

---

## 28. 初版の推奨設定

```ts
const defaultIndentSettings: IndentSettings = {
  renderMode: "typographic",
  autoInsertIndent: true,
  skipDialogueIndent: true,
};
```

初版では、行頭全角スペースを正本とし、表示上だけ `text-indent` へ変換する。

自動判定は入力補助に限定し、保存済みテキストを自動修正しない。
