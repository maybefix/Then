# brew 起動時のワークスペース復元と scratch 表示設計

起動時のワークスペース復元と scratch 表示の設計を整理して実装してください。

このアプリ brew は、基本的には「作業フォルダを開いて、その中の Markdown ファイルを編集するエディタ」として扱います。
ただし、起動時に作業フォルダが確定していない場合でも、エディタとしてすぐに書き始められる必要があります。
そのため、ユーザーに見せるメイン状態は次の2つだけにしてください。

- `workspace-ready`
- `scratch`

`no-workspace` や `workspace-missing` のような原因状態を、上部バーに直接表示しないでください。
それらは内部判定として必要であっても、ユーザー向けのメイン表示状態ではありません。

## 基本概念

### workspace-ready

作業フォルダが確定しており、その配下のファイルを開いている状態です。

例:

- 前回起動時の作業フォルダを復元できた
- ユーザーがフォルダを選択した
- `scratch` を保存し、その保存先フォルダが作業フォルダになった

上部表示:

- `作業フォルダ名 > ファイル名`

例:

- `novel > chapter01.md`
- `drafts > 剣士の目覚め.md`

### scratch

保存先未確定の無題ファイルを開いている状態です。

例:

- 初回起動で前回フォルダがない
- 前回フォルダが移動・削除・権限変更などで開けない
- ユーザーがフォルダ未選択状態から新規ファイルを作成した

上部表示:

- `一時ファイル > 無題.md`

`scratch` は、単なるエラー画面やフォルダ選択待ち画面ではありません。
実際に本文を書ける編集状態です。
起動時に作業フォルダを開けない場合も、本文エディタを表示し、`scratch` の無題ファイルを開いてください。

## 起動時の処理

起動時は、前回作業フォルダの記録を確認してください。

### 前回フォルダが存在し、読み書き可能な場合

- `workspace-ready` として開く
- 可能なら前回開いていたファイルも復元する
- 復元対象ファイルがない、または開けない場合は、その作業フォルダ内でファイル未選択または新規ファイル状態にする
- 上部表示は `作業フォルダ名 > ファイル名` にする
- アラートは出さない

### 前回フォルダの記録がない場合

- `scratch` を開く
- 上部表示は `一時ファイル > 無題.md` にする
- 本文エディタを表示し、すぐ書ける状態にする
- 「前回フォルダを開けませんでした」アラートは出さない

これは初回起動または履歴削除後の通常状態として扱います。

### 前回フォルダの記録はあるが開けない場合

例:

- フォルダが削除されている
- フォルダが移動している
- 権限がない
- 読み書き確認に失敗した

この場合も、起動直後は `scratch` を開いてください。

- 上部表示は `一時ファイル > 無題.md`
- 本文エディタを表示し、すぐ書ける状態にする
- ただし、本文エリア上部に閉じられるアラートを表示する

## 上部バーの表示方針

上部バーには、現在どこを編集しているかだけを表示してください。

表示してよいもの:

- 作業フォルダ名
- ファイル名
- 一時ファイル
- 無題.md

表示しないもの:

- 前回フォルダを開けません
- フォルダ未選択
- workspace missing
- no workspace
- 権限エラー
- 復元失敗

復元失敗や未選択の理由は、上部バーではなく、必要な場合だけ本文エリア上部のアラートで表示してください。

## アラート仕様

前回フォルダを開けなかった場合のみ、本文エリア上部にアラートを表示してください。

表示内容:

- 前回の作業フォルダを開けませんでした
- 対象パス
- 再試行
- 別のフォルダを選択
- 履歴から削除
- 閉じる

例:

```text
前回の作業フォルダを開けませんでした  
C:\Users\...\Documents\novel

[再試行] [別のフォルダを選択] [履歴から削除] [閉じる]
```

### 再試行

- 記録されている前回フォルダをもう一度確認する
- 開ける場合は `workspace-ready` に移行する
- 開けない場合は `scratch` のまま、アラートを維持する

### 別のフォルダを選択

- フォルダ選択ダイアログを開く
- 選択されたフォルダが読み書き可能なら `workspace-ready` に移行する
- `lastWorkspacePath` と `recentWorkspaces` を更新する
- アラートを閉じる

### 履歴から削除

- 開けなかった前回フォルダを `lastWorkspacePath` から削除する
- `recentWorkspaces` からも該当パスを削除する
- `scratch` のままにする
- アラートを閉じる

### 閉じる

- アラートだけ閉じる
- `scratch` は維持する
- `lastWorkspacePath` は消さない

## scratch の保存仕様

`scratch` は保存先未確定の無題ファイルです。
通常の保存操作をしたときに、作業フォルダとファイル名を確定してください。

保存時の流れ:

1. 作業フォルダが未確定なら、保存先フォルダを選択させる
2. ファイル名を入力させる
3. Markdown ファイルとして保存する
4. 保存完了後、その保存先フォルダを現在の作業フォルダとして扱う
5. `workspace-ready` に移行する
6. `lastWorkspacePath` を更新する
7. `recentWorkspaces` に追加する
8. 上部表示を `作業フォルダ名 > ファイル名` に切り替える

`scratch` 保存後も `一時ファイル > 無題.md` の表示を残さないでください。

## 新規ファイル仕様

### workspace-ready の場合

- 現在の作業フォルダ配下に新規ファイルを作成する
- ファイル名を入力させる
- 作成後、そのファイルを開く

### scratch の場合

- すでに未保存の `scratch` があるなら、破棄・保存・キャンセルの確認を行う
- 新しい `scratch` を作る場合は `一時ファイル > 無題.md` として開く
- 直接ファイルは作成しない
- 保存時に作業フォルダとファイル名を確定する

## 新規フォルダ仕様

### workspace-ready の場合

- 現在の作業フォルダ配下に新規フォルダを作成する

### scratch の場合

- 新規フォルダは直接作成しない
- フォルダには親フォルダが必要なため、先に作業フォルダを選択させる
- 作業フォルダ選択後に `workspace-ready` へ移行し、その配下で新規フォルダ作成を続行する

表示文例:

```text
フォルダを作成するには、先に作業フォルダを選択してください。
```

## recentWorkspaces

以下の場合に `recentWorkspaces` を更新してください。

- ユーザーがフォルダを選択して `workspace-ready` になった
- `scratch` を保存し、その保存先フォルダが作業フォルダになった
- 前回フォルダを復元できた

保存する情報の例:

```ts
type WorkspaceRecord = {
  path: string;
  name: string;
  lastOpenedAt: number;
};
```

同じ `path` がすでに存在する場合は重複追加せず、`lastOpenedAt` を更新してください。

## 実装上の注意

- 起動時に作業フォルダを開けない場合でも、空の選択画面だけを表示しないでください
- 「ローカル」という抽象表示を fallback として使わないでください
- ユーザーに見せるメイン状態は `workspace-ready` と `scratch` の2つにしてください
- 原因状態は上部バーではなく、必要な場合だけアラートで扱ってください
- `scratch` は本文を書ける状態です
- `scratch` は保存時に `workspace-ready` へ移行します
- 新規フォルダは `scratch` では直接作成しません
