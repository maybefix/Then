想定ユーザーシナリオ
シナリオ1: 複数の本文ファイルをB6小説PDFとして出力する

ユーザーは縦書きエディタで複数の本文ファイルを書いている。
ファイルは .txt と .md が混在している。

例:

00_扉.txt
01_第一章.txt
02_第二章.md
03_第三章.txt
99_あとがき.md

ユーザーは通常の編集画面で本文を書き終え、「エクスポート」を開く。
エクスポート画面には、出力対象ファイル一覧が表示される。

ユーザーは、出力に含めるファイルを確認し、順番を整える。
各章の開始方法を「改ページ」または「奇数ページ開始」に設定する。

ページサイズはB6を選ぶ。
余白は、天、地、ノド、小口を調整する。
ページ番号は「外側」に配置する。
ヘッダーには「現在の章タイトル」を表示する。
章扉ではヘッダーとページ番号を非表示にする。

ユーザーは「プレビュー更新」を押す。
連結後の紙面プレビューが表示される。
ユーザーはルビ、縦中横、ページ番号、改ページ位置を確認する。

問題がなければ「PDFを書き出す」を押す。
保存先を選び、PDFを生成する。

シナリオ2: A5二段組の出力に変更する

ユーザーは同じ本文ファイル群を、A5二段組でも確認したい。

エクスポート画面でページサイズをA5に変更する。
段組を2段にする。
段間、文字サイズ、行間を調整する。

ユーザーは「プレビュー更新」を押す。
紙面プレビューで、段末、段頭、改ページ、ルビ、縦中横を確認する。

読みづらい場合は、文字サイズや行間を調整して再度プレビューを更新する。
問題がなければPDFを書き出す。

このシナリオでは、設定変更とプレビュー確認が同じ画面内で完結することが重要。

シナリオ3: 出力対象から一部ファイルを外す

ユーザーは本文ファイル群の中から、あとがきだけを除外したい。

エクスポート画面で出力対象ファイル一覧を見る。
99_あとがき.md のチェックを外す。
プレビューを更新する。

プレビューには、あとがきを除いた本文だけが表示される。
ページ番号は、出力対象に含めたファイルだけで通し番号になる。

このシナリオでは、出力対象ファイル一覧が分かりやすいことが重要。

シナリオ4: 章を必ず奇数ページから始める

ユーザーは冊子向けに、各章を必ず奇数ページから始めたい。

エクスポート画面で各本文ファイルの開始方法を確認する。
01_第一章.txt、02_第二章.md、03_第三章.txt を「奇数ページ開始」に設定する。

プレビューを更新すると、必要に応じて空白ページが入り、各章が奇数ページから始まる。

このシナリオでは、ファイルごとの開始方法を一覧上で確認しやすいことが重要。

シナリオ5: ヘッダーとページ番号を調整する

ユーザーはPDFの上部に章タイトルを出し、ページ番号を下部外側に出したい。

エクスポート画面でヘッダー設定を開く。
ヘッダー内容を「現在の章タイトル」にする。
章扉ではヘッダーを非表示にする。

次にページ番号設定を開く。
ページ番号を表示にし、位置を「外側」にする。
開始番号を1にする。
章扉ではページ番号を非表示にする。

プレビューを更新し、章扉と本文ページで表示が切り替わっていることを確認する。

このシナリオでは、ヘッダー、フッター、ページ番号の設定が分かりやすくまとまっていることが重要。

シナリオ6: DOCXとして出力する

ユーザーは同じ本文ファイル群をDOCXとして出力したい。

エクスポート画面で出力形式をDOCXに切り替える。
ページサイズ、余白、フォント、段組などを確認する。
必要に応じて設定を調整する。

DOCXはPDFと完全に同じ見た目になるとは限らないが、基本的なページ設定、本文、見出し、ルビ、縦中横が反映される前提で設計する。

ユーザーは「DOCXを書き出す」を押す。
保存先を選び、DOCXを生成する。

このシナリオでは、PDFとDOCXが同じエクスポート画面から扱えることが重要。

シナリオ7: プレビュー生成に失敗する

ユーザーはエクスポート画面で「プレビュー更新」を押す。
しかし、本文変換または組版処理に失敗する。

画面には「プレビューを生成できませんでした」と表示される。
原因の種類として、次のような表示が出る。

本文の変換に失敗しました
スタイル設定の生成に失敗しました
組版処理に失敗しました
フォントが見つかりません

ユーザーは「詳細を表示」を押して、開発者向けログを確認できる。
また、「再試行」と「キャンセル」が表示される。

このシナリオでは、単にエラーコードを出すのではなく、ユーザーが次に何を確認すべきか分かる表示にすることが重要。

シナリオ8: 出力設定を保存して再利用する

ユーザーはB6小説本文用の設定を作る。
ページサイズ、余白、文字サイズ、行間、ヘッダー、ページ番号を調整する。

ユーザーは「設定保存」を押す。
設定名として「B6 小説本文」と入力する。

次回エクスポート時には、この設定を読み込める。
前回使った設定も復元できる。

このシナリオでは、毎回同じ設定を作り直さなくてよいことが重要。

シナリオ9: 狭い画面でエクスポートする

ユーザーはノートPCなどの狭い画面でエクスポート画面を開く。

広い画面では、出力対象ファイル一覧、設定パネル、プレビューを同時に表示する。
狭い画面では、タブや折りたたみで切り替える。

例:

出力対象
ページ設定
ヘッダー・フッター
プレビュー

このシナリオでは、画面が狭くてもPDF出力作業が破綻しないことが重要。

シナリオ全体で守ること
通常編集画面にPDFプレビューを常設しない
出力時だけエクスポート画面を開く
.txt と .md の混在を前提にする
「Markdownファイル」ではなく「本文ファイル」と表現する
複数ファイルを1つのPDFまたはDOCXとして出力する
ページサイズ、余白、段組、ヘッダー、フッター、ページ番号を出力直前に調整できる
プレビューは完全リアルタイムでなくてよい
最初は「プレビュー更新」ボタンで再生成する想定にする
ユーザーに外部ツールのインストールやターミナル操作を要求しない
